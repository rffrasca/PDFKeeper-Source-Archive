User FAQ

General Information

What is PDFKeeper?
It is a free, open source PDF document storage system that provides storage, indexing, and retrieval using the free Oracle Database Express Edition (Oracle Database XE).
Who should use PDFKeeper?
Anyone who wants to implement a low cost PDF document storage system in a small or home office. It can be implemented for use on a single user system or in a workgroup.

Software Compatibility

Does PDFKeeper work on other Oracle databases?
Only Oracle Database 10g Express Edition (Universal) is supported. If you are a developer, experienced with both Oracle Database and VB.NET, you should be able to get it to function, with some modification on any edition of Oracle Database 10g or later, provided Oracle Text is enabled.
Will other databases be supported in the future?
Yes, provided the database is either free or open source, and it supports full-text search and can index PDF documents. Of course, this will be dictated by available resources and demand.
Will PDFKeeper function on 64-bit versions of Windows?
64-bit has not been tested and is not supported at this time.

Troubleshooting

I get "Bad User Password" or "Password protected PDF documents are not supported by PDFKeeper" when attempting to edit the information properties of a PDF document or when attempting to upload a PDF document. What can I do?
PDFKeeper does not support PDF documents protected with a User or Owner password. If you have the legal right, you can find plenty of information on the internet to assist you. Password prompting will likely be added in a future release, for PDF documents protected with apassword.
I get "ORA-03113: end-of-file on communication channel" following the upload of a PDF document. What can I do?
If your database schema was built using the Database Setup, included with PDFKeeper 1.0.0 or 1.1.0, please refer to this alert for the workaround.
