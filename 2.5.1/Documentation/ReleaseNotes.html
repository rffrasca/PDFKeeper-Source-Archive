Introduction

This document contains the release notes for PDFKeeper version 2.5.1. The following sections describe the release in detail and provide late-breaking or other information that supplements the main documentation.

What's New?

  • This is a bug fix release to address critical defects with the Information Properties Editor Folder Watcher.
  • The source code for this release is available at website.
  • Improvements to the product's quality and reliability. See "Recent Changes" below for details.

Recent Changes

  • FIX 104 Information Properties Editor Folder Watcher detecting duplicate events.
  • FIX 105 Information Properties Editor Folder Watcher launching Information Properties Editor while PDF is being written.
  • FIX 106 Folder Watcher log file only contains the last error logged.
  • FIX 107 Folder Watcher Unhandled Exception: System.ArgumentOutOfRangeException: startIndex can not be less than zero.

Upgrade Notes

Deprecated Components Information

Since PDFKeeper 2.0.0, the server-side components have been deprecated. If not upgrading a system that has only the server-side components from PDFKeeper 1.0.0 or 1.1.0 installed, please uninstall PDFKeeper from that system. From the server; remove the Document Loader scheduled task, Document Upload Folder, and delete the PDFKEEPER_UPLOAD environment variable. From each client, remove the shortcut to the Document Upload folder.

Database Changes
In PDFKeeper 1.0.0 and 1.1.0, the Oracle Text WORLD_LEXER was used to provide broad language support during PDF document indexing. However, a bug exists with this lexer in the Oracle Database 10g Express Edition, see Issue 50 for details. As a result of this bug, starting with PDFKeeper 2.0.0, the BASIC_LEXER is now used. This change will only affect a database schema built in PDFKeeper 2.0.0 or later; an existing database schema will need to be altered if an ORA-03113 error is encountered while uploading documents.
For more information on the BASIC_LEXER, including the languages supported, please refer to the Oracle Text Documentation.

Known Problems and Workarounds

  • ORA-03113: end-of-file on communication channel following the upload of a PDF document and your database schema was built using the Database Setup included with PDFKeeper 1.0.0 or 1.1.0.
    • WORKAROUND: Please refer to the alert for the workaround.
